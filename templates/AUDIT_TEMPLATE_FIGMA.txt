HOW TO USE THIS AUDIT TEMPLATE

This template is built as a selectable audit system.

For each section:
1. Review the site externally using Lighthouse/PageSpeed, DevTools, page source, sitemap, search results, and the live website.
2. Under the main issue list, keep only the bracketed issues that apply to the site.
3. Under “The Affects”, keep only the effects that match the selected issue IDs.
4. Under “The Fixes”, keep only the fixes that match the selected issue IDs.
5. Remove anything that is not relevant so the final audit stays short, clear, and specific.

The numbers in brackets show which issue each effect or fix relates to.

In the finished audit, keep selected issues in bracket form:

[2] The main visible content takes too long to load.

Do not turn selected issues into a normal numbered list, because skipped issue numbers look broken to the reader.

Use plain-language performance notes in the finished audit. Avoid raw technical measurements, platform names, hosting names, or file-size units unless the user asks for technical detail.

Important:
Anything inside square brackets must be replaced with real audit content.

Example:
[COMPANY NAME] becomes Rides2U
[WEBSITE URL] becomes https://www.rides2u.com/
[Score] becomes the actual Speed test score
[Main loading issue / note] becomes the real issue found during the audit

Do not leave square bracket placeholders in the final client audit. Bracketed issue IDs such as [2] are allowed.
